# Дорожная карта labbooking

> Этапы 0–3 реализованы в коде. Этап 4 (SSO API, React SPA) — заглушки и CSV-импорт Деканата.

Опорный проект: **labbooking** (Django + DRF + PostgreSQL + Docker).  
Цель ближайшего этапа: **рабочий сайт на VM в Docker**, затем доработка по ТЗ СПГУ.

---

## Режим работы (работа / дом)

| Где | Что делаете |
|-----|-------------|
| **Работа** | Код в Cursor, `git commit/push`, чтение ТЗ, планирование |
| **Дом** | `git pull`, тесты, `bash scripts/deploy-vm.sh`, SSH на VM |
| **С работы** | Только браузер: `http://<IP>/` (без SSH) |

---

## Этап 0 — Сегодня вечером: деплой на VM

**Цель:** сайт открывается по публичному IP.

| Шаг | Действие | Документ |
|-----|----------|----------|
| 1 | Push кода в Git | — |
| 2 | SSH на VM, установить Docker | [DEPLOY_YANDEX_VM.md](DEPLOY_YANDEX_VM.md) |
| 3 | `cp .env.vm.example .env`, указать IP и пароли | `.env.vm.example` |
| 4 | `bash scripts/deploy-vm.sh` | `scripts/deploy-vm.sh` |
| 5 | Проверить в браузере | — |

**Критерий готовности:** health OK, вход студентом, запись на ЛР.

| Из ТЗ | Задача |

## Рекомендации по моделям AI (Cursor)

| Тип задачи | Модель | Режим |
|------------|--------|-------|
| Деплой, Docker, `.env`, скрипты | **Composer Fast** | Agent |
| План этапа, gap-анализ ТЗ | **GPT-5.5 Medium** / **Opus 4.8** | Plan / Ask |
| Django: модели, `BookingService`, API | **GPT-5.3 Codex** | Agent |
| Расписание, Деканат, SSO (архитектура) | **Opus 4.8** → затем Codex | Plan → Agent |
| Мелкие правки, тесты, lint | **Composer Fast** | Agent |
| Security перед prod | **Opus 4.8** + `.cursor/rules/security.md` | Ask |

**Правило:** один чат = одна фича; в начале чата: `@docs/ROADMAP.md` `@docs/zavlab_tz.txt`.
